# Purpose

This guide defines the correct import sequence for all datasets generated by the Enterprise Synthetic Data Generation Engine (ESDGE).

The import order preserves foreign key relationships and ensures consistent database seeding.

# Notes

- Import datasets only in the order listed above.
- Never skip a parent dataset.
- Preserve generated primary keys.
- Preserve generated foreign keys.
- Do not manually edit CSV identifiers before importing.
